阶段性科研汇报：MultiPref 分歧偏好 baseline 复现

一、目前在做什么

我这段时间主要在复现论文 Diverging Preferences: When do Annotators Disagree and do Models Know? 里和 single-value reward model 有关的 baseline 评估流程。

我关注的问题是：当不同标注者对同一个 preference pair 有分歧时，reward model 的分数差是否也会变小。换句话说，模型是否能通过自己的 reward gap 反映出“这个样本本来就容易让人产生不同偏好”。

目前我没有训练新模型，只做现有 reward model 的 inference 和评估。


二、目前完成的实验

数据集用的是 MultiPref。模型用的是 Skywork-Reward-Llama-3.1-8B-v0.2。

由于本地显存有限，我没有跑论文里的 Skywork 27B，也没有做完整 MultiPref 全量评估。目前的结果是基于本地 8GB 显存下的小规模复现，主要目的是先把 baseline pipeline 跑通，并看 reward gap 是否有基本信号。

具体做法是：对每个 prompt 和两个候选回答 A、B，分别用 reward model 打分，得到 score_a 和 score_b。

如果 score_a 大于 score_b，就认为模型偏好 A；如果 score_b 大于 score_a，就认为模型偏好 B。然后把模型预测结果和人类多数偏好进行比较，得到 preference accuracy。

另外，我用 abs(score_a - score_b) 表示 reward gap。gap 越大，说明模型越确定 A 和 B 之间有明显差别；gap 越小，说明模型对两个回答的偏好没那么确定。


三、Preference Accuracy 结果

我在 100 条有明确人类多数偏好的随机样本上做了评估。

结果是：

Preference Accuracy = 0.7500
Majority-class baseline = 0.6600

也就是说，在这个 sampled subset 上，Skywork 8B 的表现高于简单地一直预测多数类别的 baseline。

这个结果只能说明模型在小样本上有一定恢复人类多数偏好的能力，不能直接等同于论文的完整结果。


四、Diverging ID 结果

接着我评估 reward gap 是否能区分“人类有分歧的样本”和“人类高度一致的样本”。

这里我把 paper-like diverging examples 当作正类，把 high-agreement preference examples 当作负类。然后用 -abs(score_a - score_b) 作为 diverging score。

这样定义的原因是：gap 越小，说明模型越不确定；如果一个样本本身更容易让人类产生分歧，理论上 reward model 的 gap 也可能更小。所以用负的 gap 作为“更像 diverging”的分数。

结果是：

Diverging ID AUROC = 0.6244

reward gap 的对比如下：

Diverging examples:
mean gap = 10.1316
median gap = 7.3623

High-agreement examples:
mean gap = 14.6600
median gap = 11.9885

这个结果说明，diverging examples 的 reward gap 确实比 high-agreement examples 更小。也就是说，reward gap 对识别人类偏好分歧有一定信号。不过 AUROC 只有 0.6244，说明这个信号不算强，只能说是弱到中等程度。


五、定性 case study

除了看指标，我还抽了两类样本做人工检查。

第一类是 high-confidence preference errors。也就是 Skywork 和人类多数偏好不一致，并且 reward gap 还比较大的样本。这类样本说明模型不只是错了，而且是比较确定地错了。

第二类是 diverging examples with large reward gaps。也就是人类标注者本来有分歧，但 Skywork 仍然给出了很大的 A/B 分数差。这类样本说明，single-value reward model 有时会把本来多维度的偏好分歧压缩成一个很确定的标量判断。

我把这些 case 粗略分了几类。总体上，出现最多的是：

verbosity / detail / concision 相关问题：8 条
safety / refusal / redirection 相关问题：4 条
technical / medical complexity 相关问题：3 条
task ambiguity 相关问题：2 条
factuality / instruction-following / accuracy 相关问题：1 条
sensitive topic framing 相关问题：1 条
other：1 条

比较明显的现象是，模型有时会偏好更长、更详细或更正式的回答，但人类不一定总是这么判断。还有一些 safety case 中，模型会在直接拒绝和安全引导之间表现出和人类不同的偏好。对于一些任务本身比较模糊的样本，不同回答在不同理解下都说得通，这时单一 reward score 很难完全表达这种分歧。


六、目前的主要结论

目前这个小规模复现得到的结论比较有限，但有几个初步观察。

第一，Skywork 8B 在 sampled subset 上可以超过简单 majority-class baseline，说明它有一定恢复人类多数偏好的能力。

第二，reward gap 对识别人类分歧有一定信号。diverging examples 的平均 gap 和中位数 gap 都比 high-agreement examples 更小。

第三，这个信号并不强。AUROC 只有 0.6244，说明 reward gap 不能稳定地区分所有 diverging 和 high-agreement 样本。

第四，从 case study 看，single-value reward model 在一些多维偏好冲突中仍然会给出很确定的判断。尤其是在详细程度、事实性、指令遵循、安全引导和任务歧义这些维度发生冲突时，模型的偏好可能和人类分布不完全一致。


七、局限性

目前这个实验不是论文 Table 3 的严格复现。

主要限制有：

1. 我用的是 Skywork 8B，不是论文里的 Skywork 27B。
2. 由于本地显存限制，使用了 4-bit inference。
3. 目前只在 sampled subset 上评估，没有跑完整 MultiPref。
4. 为了避免显存问题，过滤了较长样本。
5. case study 的规模还比较小，人工标注也比较初步。

所以目前更准确的定位是：在本地算力限制下，对 single-value reward model baseline evaluation pipeline 的一个小规模复现和初步分析。


八、下一步想法


接下来我想尝试构造一个 no-training 的 preference pair diagnostic framework。具体来说，可以基于 annotator vote distribution 和文本表面特征，判断一个 pair 更接近 clear preference、genuine disagreement、ambiguous pair，还是 potentially low-quality pair。

可能使用的特征包括 annotation entropy、majority strength、tie rate、slight preference rate、response length imbalance、format mismatch、refusal mismatch、caution mismatch 等。目标不是预测哪个 response 更好，而是判断这个 preference pair 本身是否清晰、是否有歧义、是否需要人工复核。
